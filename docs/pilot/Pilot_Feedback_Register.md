## 处理规则

### 接受条件

仅满足以下全部条件的反馈可进入修复：

1. ✅ 来自真实用户（非假设场景）
2. ✅ 来自真实业务数据操作（非 Demo 数据）
3. ✅ 属于三类之一（业务流断点 / 理解问题 / 效率问题）

### 修复范围限制

| 允许 | 禁止 |
|:-----|:-----|
| 修复按钮/链接失效 | 新增页面 |
| 修复数据展示错误 | 新增数据库字段 |
| 修复筛选/分页异常 | 新增 API 端点 |
| 优化已存在流程的体验 | 新增业务功能 |
| 修复 Console/API 报错 | UI 设计重构 |

### 每周汇总

每周末执行：
1. 整理本周所有反馈，按分类排序
2. 标注严重度：P0(影响业务)、P1(重要体验)、P2(后续版本)
3. P0 立即修复，P1 进入当周 Sprint，P2 进入 Backlog
4. 输出《Pilot Week N Summary》
