# Master Data Specification — FinanceDesk V1

> PS-002 文档，定义 FinanceDesk 主数据的来源、维护责任和更新机制。面向财务、项目经理和 IT 运维人员。

---

## 1. 总则

FinanceDesk 的 5 类核心主数据：

| 数据 | 来源 | 维护责任 | 更新时机 | 手动/自动 |
|------|------|----------|----------|-----------|
| 合同 | 人工录入 + ERP 补充 | 财务 | 合同签订时 | 手动 |
| 项目 | 自动派生自合同 | 系统 + 项目经理 | 签单后自动创建 | 自动 |
| 订单 | 人工录入 | 项目经理 | 开工前 | 手动 |
| 客户 | 从合同自动提取 | 系统 | 合同录入后 | 自动 |
| 供应商 | 人工维护 | 采购/财务 | 合同签订时 | 手动 |

---

## 2. 合同（Contract / Project）

### 2.1 定义

合同是 FinanceDesk 的核心业务单元，对应 ERP 中的项目。系统使用 `Project` 模型存储合同信息。

### 2.2 字段定义

| 字段 | 类型 | 必填 | 来源 | 说明 |
|------|------|------|------|------|
| 合同名称 | String(200) | 是 | 人工录入 | 框架合同名称或单项合同名称 |
| 合同编号 | String(100) | 是 | 人工录入 | 新增必填，历史可为空 |
| 合同类型 | String(20) | 是 | 人工选择 | 单项合同 / 框架合同 |
| 合同金额 | Numeric(15,2) | 是 | 人工录入 | 含税合同总金额 |
| 签订日期 | Date | 推荐 | 人工录入 |    |
| 开始日期 | Date | 推荐 | 人工录入 |    |
| 结束日期 | Date | 推荐 | 人工录入 |    |
| 负责人 | String(100) | 推荐 | 人工录入 | 项目经理 |
| 所属部门 | String(100) | 推荐 | 人工录入 |    |
| 业主单位 | String(200) | 推荐 | 人工录入 |    |
| ERP 编号 | String(100) | 可选 | ERP 导入 | 只读，由 ERP 匹配后反写 |
| 合同状态 | String(50) | 系统推导 | 自动 | 待执行 / 进行中 / 已完成 / 已终止 |

### 2.3 录入规范

```
合同编号格式: CT-{YYYY}-{NNN}
示例: CT-2026-001, CT-2026-002
```

- 编号由系统自动生成（新增时）
- 合同类型选择"框架合同"时，可关联多个订单
- 合同类型选择"单项合同"时，通常只关联一个订单

### 2.4 维护指引

| 场景 | 操作 | 责任人 |
|------|------|--------|
| 新签合同 | 录入合同基本信息 | 财务 |
| 合同变更 | 修改合同金额、日期等 | 财务 |
| 合同终止 | 将状态改为"已终止" | 财务 |
| ERP 编号关联 | 在导入匹配时自动或人工关联 | 系统/财务 |

---

## 3. 项目（Project — 经营维度）

### 3.1 定义

项目在 FinanceDesk 中的定义与合同相同（同一张 `Project` 表）。一个合同 = 一个项目。在 Dashboard 中以"项目"维度展示。

> 当前 V1 阶段不区分"框架合同下的子项目"。如有需求将在 V2 中引入层级结构。

### 3.2 创建方式

```
合同录入
    ↓
系统自动创建项目
    ↓
项目名称 = 合同名称
    ↓
可在 Dashboard 中按项目维度分析
```

### 3.3 维护指引

| 场景 | 操作 | 责任人 |
|------|------|--------|
| 项目创建 | 通过合同录入自动创建 | 系统 |
| 项目信息变更 | 修改对应合同信息 | 财务 |
| 项目关闭 | 所有订单完成后自动推导 | 系统 |

---

## 4. 订单（Order）

### 4.1 定义

订单是 FinanceDesk 的最小经营单位。一个合同（项目）下可以有多个订单。

订单对应：
- **收入端**: 合同下的具体施工/设备/服务项目
- **成本端**: 每个订单需要采购的材料、分包、劳务等

### 4.2 字段定义

| 字段 | 类型 | 必填 | 来源 | 说明 |
|------|------|------|------|------|
| 订单编号 | String(100) | 是 | 人工录入 | 人工指定，不自动编号 |
| 订单名称 | String(200) | 是 | 人工录入 | 明确的业务名称 |
| 所属项目 | FK → Project | 是 | 选择 | 关联到已录入的合同 |
| 订单金额 | Numeric(15,2) | 是 | 人工录入 | 含税金额 |
| 负责人 | String(100) | 推荐 | 人工录入 |    |
| 订单类型 | String(50) | 推荐 | 人工录入 | 工程施工/设备采购/服务等 |
| 订单状态 | 系统推导 | 自动 | 自动 | 待开票/正常/已完成 |

### 4.3 录入规范

**订单编号规则**

订单编号**人工指定**（不自动生成），因为用户明确要求人工控制编号规则：

```
项目经理习惯的编号方式，如：
CG-2026-001（采购单号）
SG-2026-001（施工单号）
或直接使用 ERP 中的订单号
```

**金额分配规则**

```
合同总金额 ≥ 所有订单金额之和
允许多个订单金额总和 < 合同金额（剩余部分为预留）
订单金额不可超过合同金额
```

### 4.4 维护指引

| 场景 | 操作 | 责任人 |
|------|------|--------|
| 新订单 | 录入订单，关联到项目 | 项目经理 |
| 订单变更 | 修改订单金额/信息 | 项目经理 |
| 订单关闭 | 收入=订单金额且成本=已确认 | 系统推导 |
| 订单删除 | 逻辑删除（标记 is_deleted） | 项目经理 |

---

## 5. 客户（Customer）

### 5.1 定义

客户是合同的甲方（业主方），从合同信息中自动提取和归集。

### 5.2 数据来源

| 数据 | 来源 |
|------|------|
| 客户名称 | 合同的「业主单位」字段 |
| 联系人 | 合同的「联系人」字段 |
| 联系电话 | 合同的「联系电话」字段 |

### 5.3 维护指引

| 场景 | 操作 | 责任人 |
|------|------|--------|
| 客户信息变更 | 在对应合同中修改 | 财务 |
| 重复客户合并 | 联系 IT 执行数据库操作 | IT |

> V1 阶段客户不是独立实体（无独立客户表）。客户信息分散在各合同中。如有独立客户管理的需求，将在 V2 中引入 `Customer` 表。

---

## 6. 供应商（Vendor / Supplier）

### 6.1 定义

供应商是 FinanceDesk 成本端的外部合作方，包括分包商、材料供应商、劳务公司等。

### 6.2 数据模型

供应商分为两个层级：

| 表 | 用途 |
|----|------|
| `Vendor` | 供应商主体信息（名称、账户、联系人等） |
| `Supplier` | 供应商资质/证照信息 |
| `SupplierPrice` | 供应商报价（针对具体品类） |
| `SupplierYearPrice` | 供应商年度价格 |
| `SupplierContract` | 与供应商签订的成本合同 |
| `SupplierUnitPrice` | 供应商单价管理 |

### 6.3 字段定义（Vendor 主体）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 供应商名称 | String(200) | 是 | 营业执照上的全称 |
| 统一社会信用代码 | String(50) | 推荐 | 工商注册号 |
| 银行账户 | String(100) | 是 | 付款用 |
| 开户行 | String(200) | 推荐 |    |
| 联系人 | String(100) | 推荐 |    |
| 联系电话 | String(50) | 推荐 |    |
| 地址 | String(500) | 可选 |    |

### 6.4 录入规范

```
供应商编号: 系统自动生成
```

- 一个供应商只需录入一次基本信息（Vendor）
- 各业务的合同/报价分别维护（SupplierContract / SupplierPrice）

### 6.5 维护指引

| 场景 | 操作 | 责任人 |
|------|------|--------|
| 新供应商 | 录入主体信息 + 资质信息 | 采购 |
| 供应商变更 | 修改对应信息 | 采购 |
| 年度报价更新 | 更新 SupplierYearPrice | 采购 |
| 供应商停用 | 逻辑删除 | 采购 |

## 7. 数据质量要求

### 7.1 必填字段缺失率

| 数据 | 目标值 |
|------|--------|
| 合同必填字段完整率 | ≥ 95% |
| 订单必填字段完整率 | ≥ 90% |
| 供应商必填字段完整率 | ≥ 80%（上线阶段） |

### 7.2 数据一致性

```
合同金额 = 所有订单金额之和 + 预留金额
订单金额 ≥ 累计收入 + 收入 Gap
供应商付款 ≤ 供应商合同金额
```

### 7.3 清理机制

- 逻辑删除（`is_deleted` 标记），不物理删除
- 删除的数据可在 30 天内恢复
- 超过 30 天的逻辑删除数据归档到历史表
