# Import / Export Specification — FinanceDesk V1

> PS-002 文档，定义 FinanceDesk 的导入/导出模板、编码规则和归档规范。面向财务操作人员。

---

## 1. 导入模板

### 1.1 收支明细模板

文件: `template_收支明细.xlsx`

| 列名 | 类型 | 必填 | 示例 | 说明 |
|------|------|------|------|------|
| 凭证日期 | 文本(YYYY-MM-DD) | 是 | 2026-06-15 | 财务凭证日期 |
| 凭证号 | 文本 | 是 | PZ-2026-0615-001 | 唯一标识，防重复 |
| 摘要 | 文本 | 推荐 | 崇左移动项目-设备采购 | 模糊匹配关键词来源 |
| 客户名称 | 文本 | 推荐 | 崇左移动 | 匹配项目用 |
| 借方金额 | 数字(2位小数) | 条件 | 50000.00 | 支出方向 |
| 贷方金额 | 数字(2位小数) | 条件 | 0.00 | 收入方向 |

> 条件：借方和贷方至少有一方有值，不可同时为空。

### 1.2 应收明细模板

文件: `template_应收明细.xlsx`

| 列名 | 类型 | 必填 | 示例 | 说明 |
|------|------|------|------|------|
| 开票日期 | 文本(YYYY-MM-DD) | 是 | 2026-06-20 | 发票开具日期 |
| 发票号码 | 文本 | 是 | INV-2026-0615-001 | 发票号，防重复 |
| 项目名称 | 文本 | 是 | 崇左移动一期工程 | 匹配项目用 |
| 开票金额 | 数字(2位小数) | 是 | 500000.00 | 含税金额 |

### 1.3 收款明细模板

文件: `template_收款明细.xlsx`

| 列名 | 类型 | 必填 | 示例 | 说明 |
|------|------|------|------|------|
| 到账日期 | 文本(YYYY-MM-DD) | 是 | 2026-06-25 | 银行到账日期 |
| 银行回单号 | 文本 | 是 | BANK-2026-0625-001 | 回单号，防重复 |
| 付款单位 | 文本 | 是 | 崇左移动 | 匹配项目用 |
| 收款金额 | 数字(2位小数) | 是 | 300000.00 | 实际到账金额 |

## 2. 模板下载

用户可以在 FinanceDesk 的 ERP 导入页面点击「下载模板」获取标准模板：

```
ERP 导入页面
    ├── 📥 下载收支明细模板
    ├── 📥 下载应收明细模板
    └── 📥 下载收款明细模板
```

所有模板均为 `.xlsx` 格式，包含列名和示例数据。

## 3. 导出模板

### 3.1 月度经营报表

| 列名 | 说明 | 来源 |
|------|------|------|
| 合同编号 |    | Project.contract_no |
| 合同名称 |    | Project.framework_name |
| 负责人 |    | Project.manager |
| 合同金额 |    | Project.contract_amount |
| 订单数 |    | COUNT(Order.id) |
| 累计收入 |    | SUM(IncomeFlow.taxable_amount) |
| 累计成本 |    | SUM(CostFlow.taxable_amount) |
| 毛利 | 收入 - 成本 | 计算 |
| 毛利率 | 毛利/收入 × 100% | 计算 |
| 累计回款 |    | SUM(Collection.amount) |
| 累计付款 |    | SUM(Payment.amount) |
| 应收余额 | 收入 - 回款 | 计算 |
| 应付余额 | 成本 - 付款 | 计算 |
| 经营完成率 | 回款/收入 × 100% | 计算 |

### 3.2 订单明细导出

| 列名 | 说明 |
|------|------|
| 订单编号 | Order.order_no |
| 订单名称 | Order.order_name |
| 所属项目 | Order.project_id → Project.framework_name |
| 订单金额 | Order.amount |
| 累计收入 | SUM(IncomeFlow.taxable_amount) |
| 累计成本 | SUM(CostFlow.taxable_amount) |

| 累计回款 | SUM(Collection.amount) |
| 累计付款 | SUM(Payment.amount) |

## 4. 编码规则

### 4.1 系统编码

| 实体 | 编码格式 | 示例 | 生成方式 |
|------|----------|------|----------|
| 合同编号 | CT-{YYYY}-{NNN} | CT-2026-001 | 系统自动，按年顺序 |
| 导入批次 | IMP-{YYYYMMDD}-{NNN} | IMP-20260705-001 | 系统自动，当日顺序 |
| 备份文件 | fd_backup_{YYYYMM}.json | fd_backup_202606.json | 系统自动 |

### 4.2 人工编码

| 实体 | 规则 | 示例 | 说明 |
|------|------|------|------|
| 订单编号 | 人工指定 | CG-2026-001 | 项目经理习惯的编号 |
| 供应商编号 | 系统自动 | V-{NNNN} | 按录入顺序递增 |
| 成本合同编号 | 人工指定 | 按内部规则 | 可沿用采购系统编号 |

## 5. 月度归档

### 5.1 归档目录结构

```
/var/backups/financedesk/
    ├── 2026/
    │   ├── 06_六月/
    │   │   ├── fd_backup_202606.json          ← 全量数据备份
    │   │   ├── erp_202606_收支.xlsx            ← 导入源文件
    │   │   ├── erp_202606_应收.xlsx
    │   │   ├── erp_202606_收款.xlsx
    │   │   └── reconciliation_202606.xlsx     ← 月度对账表
    │   └── 07_七月/
    │       └── ...
    └── 2027/
        └── ...
```

### 5.2 归档流程

```
每月 5 日（关账后）
    │
    ├── ① 系统自动导出全量数据为 JSON
    │
    ├── ② 财务手动保存当月导入的 ERP 源文件
    │
    ├── ③ 财务制作月度对账表（ERP vs FinanceDesk）
    │
    └── ④ 所有文件存入月度归档目录
```

### 5.3 保留策略

| 数据类型 | 保留期限 | 说明 |
|----------|----------|------|
| 月度全量备份 | 12 个月 | 超过 12 个月自动删除最旧备份 |
| ERP 导入源文件 | 12 个月 | 财务自行保留，系统不强制 |
| 月度对账表 | 3 年 | 审计要求 |

## 6. 数据版本管理

### 6.1 版本标识

每次确认导入生成唯一的 `ImportBatch`，作为该次操作的数据版本标识。

```json
{
  "batch_no": "IMP-20260705-001",
  "version_label": "2026年6月 第1次导入"
}
```

### 6.2 版本回溯

通过 `ImportBatch` 可以追溯到：

```
某一笔经营数据
    ↓
属于哪个 ImportBatch
    ↓
当时导入的 ERP 源文件（文件在归档目录中）
    ↓
ERP 系统的原始凭证
```

### 6.3 版本冲突

| 场景 | 处理方式 |
|------|----------|
| 同一凭证号重复导入 | 系统自动跳过（唯一约束），记录重复数 |
| 已关账月份的新数据 | 作为补充批次导入（batch_no 独立） |
| 修正已导入的数据 | 在业务表直接修正，审计日志记录变更 |
