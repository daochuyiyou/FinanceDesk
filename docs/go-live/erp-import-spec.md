# ERP Import Specification — FinanceDesk V1

> PS-002 文档，定义 ERP 数据导入 FinanceDesk 的完整规范。面向财务操作人员和系统开发人员。

---

## 1. 支持的 ERP 导出文件

### 1.1 文件格式

| 属性 | 要求 |
|------|------|
| 格式 | **Excel (.xlsx)** — 不支持 .xls、.csv |
| 字符编码 | UTF-8（导出时指定，避免乱码） |
| 文件大小 | ≤ 10 MB（超过请联系 IT 分批导出） |
| 工作表命名 | `收支明细`、`应收明细`、`收款明细`（每个 Sheet 对应一种记录类型） |

> 中移建设崇左分公司当前 ERP 系统支持直接导出 .xlsx 格式。如导出为 .xls，请在导出时选择「另存为 .xlsx」。

### 1.2 文件命名规范

```
{导出期间}_{月份序号}_{操作类型}_{ERP系统名}.xlsx
```

**示例：**

| 文件名 | 含义 |
|--------|------|
| `2026-06_01_收支_ERP.xlsx` | 2026年6月第1次导出，收支明细 |
| `2026-06_02_应收_ERP.xlsx` | 2026年6月第2次导出，应收明细 |
| `2026-06_03_收款_ERP.xlsx` | 2026年6月第3次导出，收款明细 |

**规则：**
- 月份序号：同一月份多次导出时递增（01 → 02 → 03）
- 操作类型：`收支` / `应收` / `收款`
- 所有文件后缀为 `.xlsx`

### 1.3 导出频率

| 场景 | 频率 | 说明 |
|------|------|------|
| 月度例行 | 每月 5 日前 | 财务导出上月全量收支/应收/收款数据 |
| 补充数据 | 按需 | 当月内有新增业务或补录发票时 |
| 月末关账 | 每月最后工作日 | 确保当月数据完整 |

---

## 2. 导入顺序

```
① 收支明细（income_expense）
    ↓
② 应收明细（receivable）
    ↓
③ 收款明细（collection）
```

**不可颠倒导入顺序**，因为应收和收款数据依赖收支数据中的凭证号进行去重和匹配。

---

## 3. 字段映射

### 3.1 收支明细 → ERPStagingFlow (record_type=income_expense)

| Excel 列名 | 目标字段 | 类型 | 必填 | 说明 |
|-----------|---------|------|------|------|
| 凭证日期 | `occur_date` | Date | 是 | 格式 YYYY-MM-DD |
| 凭证号 | `erp_record_id` | String(100) | 是 | 唯一约束，防重复导入 |
| 摘要 | `summary` | Text | 推荐 | 核心模糊匹配字段 |
| 客户名称 | `raw_project_name` | String(300) | 推荐 | 用于匹配 FinanceDesk 项目 |
| 借方金额 | `amount_out` | Numeric(15,2) | 条件 | 支出/成本方向 |
| 贷方金额 | `amount_in` | Numeric(15,2) | 条件 | 收入方向 |
| 来源文件 | `source_file` | String(500) | 自动 | 系统自动记录 |

### 3.2 应收明细 → ERPStagingFlow (record_type=receivable)

| Excel 列名 | 目标字段 | 类型 | 必填 | 说明 |
|-----------|---------|------|------|------|
| 开票日期 | `occur_date` | Date | 是 | 格式 YYYY-MM-DD |
| 发票号码 | `erp_record_id` | String(100) | 是 | 发票号，防重复 |
| 项目名称 | `raw_project_name` | String(300) | 是 | 匹配项目用 |
| 开票金额 | `amount_in` | Numeric(15,2) | 是 | 含税金额 |

### 3.3 收款明细 → ERPStagingFlow (record_type=collection)

| Excel 列名 | 目标字段 | 类型 | 必填 | 说明 |
|-----------|---------|------|------|------|
| 到账日期 | `occur_date` | Date | 是 | 格式 YYYY-MM-DD |
| 银行回单号 | `erp_record_id` | String(100) | 是 | 回单号，防重复 |
| 付款单位 | `raw_project_name` | String(300) | 是 | 匹配项目用 |
| 收款金额 | `amount_in` | Numeric(15,2) | 是 | 实际到账金额 |

## 4. 校验规则

### 4.1 导入前校验（服务端自动）

| # | 规则 | 失败处理 |
|---|------|----------|
| 1 | 文件格式必须是 .xlsx | 拒绝导入，提示"仅支持 .xlsx 格式" |
| 2 | 文件大小 ≤ 10 MB | 拒绝导入，提示"文件过大，请分批导出" |
| 3 | 必要列必须存在 | 标记该行为错误，显示缺失列名 |
| 4 | 必填字段不能为空 | 标记该行为错误，提示"第 X 行 XX 字段缺失" |
| 5 | 金额字段必须为数字 | 标记该行为错误，提示"第 X 行金额格式错误" |
| 6 | 日期字段必须为有效日期 | 标记该行为错误，提示"第 X 行日期格式错误" |
| 7 | 凭证号/发票号唯一 | 标记为重复，跳过导入 |
| 8 | 借贷方至少有一方有值 | 标记该行为错误，提示"金额缺失" |

### 4.2 导入后校验（人工在 Sandbox 确认）

| # | 校验项 | 说明 |
|---|--------|------|
| 1 | 总数核对 | 导入条数 ≈ ERP 导出条数（允许重复跳过） |
| 2 | 金额汇总 | 导入金额合计 ≈ ERP 导出金额合计 |
| 3 | 未匹配检查 | 匹配状态为 `pending` 的记录需要人工归集 |
| 4 | 异常记录 | 自动匹配异常的记录需要人工确认 |

### 4.3 匹配规则

**自动匹配顺序（MatchingEngine）：**

```
① 精确匹配: erp_record_id == order_no 或 invoice_no
② 关键词匹配: summary 或 raw_project_name 包含 ProjectKeywordMapping.keyword
③ 模糊匹配: raw_project_name LIKE '%keyword%'
```

未匹配的记录标记为 `match_status=pending`，由人工在 Workbench 界面处理。

## 5. 错误处理

### 5.1 错误分类

| 类别 | 示例 | 处理方式 |
|------|------|----------|
| **格式错误** | 文件不是 .xlsx、列缺失 | 拒绝导入，返回明确错误信息 |
| **数据错误** | 金额格式错误、日期无效 | 跳过该行，记录到 `ImportBatch.failed_count` |
| **重复数据** | 凭证号已存在 | 跳过该行，记录到 `ImportBatch.duplicate_count` |
| **匹配失败** | 找不到对应项目/订单 | 标记 `match_status=pending`，人工处理 |

### 5.2 反馈机制

导入完成后系统自动返回：

```json
{
  "batch_no": "IMP-20260705-001",
  "total": 150,
  "success": 145,
  "failed": 2,
  "duplicate": 3,
  "pending_match": 10
}
```

财务操作人员在导入结果页面可以：
- 查看失败记录的详细错误原因
- 下载失败记录为 Excel（供修正后重新导入）
- 查看待匹配记录并在人工匹配界面处理

### 5.3 回滚机制

**确认导入前（Sandbox 阶段）：** 可以清除本次导入的所有暂存数据，重新导入。

**确认导入后（已写入业务表）：** 不支持自动回滚。需要手动在业务表中修正数据。误导入的处理流程：

```
① 联系 IT 确认影响范围
② 在 ImportBatch 记录中标记为"已回滚"
③ 手动修正业务表数据
④ 重新导入正确的数据
```

## 6. 月度导入流程（标准流程）

```
每月 5 日前
    │
    ├── ① 财务从 ERP 导出 3 个 Sheet（收支、应收、收款）
    │
    ├── ② 上传至 FinanceDesk → Sandbox（数据暂存，不写入业务表）
    │
    ├── ③ 在 Sandbox 检查：
    │   ├── 总数核对
    │   ├── 金额合计核对
    │   └── 未匹配记录处理
    │
    ├── ④ 确认导入 → 写入业务表
    │
    └── ⑤ 检查 Dashboard 数据是否与 ERP 同期数据一致
```
