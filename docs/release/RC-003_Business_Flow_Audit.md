## 发现的业务流断裂点

### F1: 合同→订单（🎯 最高优先级）

**位置**: `ContractDrawer.tsx:70`

```tsx
{ title: '订单编号', dataIndex: 'order_no',
  render: (v: string) => <a>{v}</a>,  // ← 呈现为链接但无onClick！
},
```

用户看到可点击的链接样式，**但点下去无任何反应**。这是 PS-002 预留功能未实现。

**影响**: 用户在合同 Drawer 中看到订单列表，想查看某订单的完整经营情况→无法点击→只能记住订单号→菜单切页→搜索→才能看到。

**修复**: 加 `onNavigate` + `onClick`。纯 UI 逻辑，零维护。

---

### F2: 订单管理 KPI 不能跳转

**位置**: `OrderBusinessWorkbench.tsx:159-168`

8 个 KPI 卡片全部使用 `() => message.info('累计收入')`，点了只弹 toast，不导航。

**影响**: 用户看到"累计收入 ¥100万"，想查看明细→点不下去→只能切换菜单到收入页面。

**修复**: KPI 点击改为 `onNavigate` 跳转对应页面。Order 已有 `onNavigate` prop。

| KPI | 应跳转 |
|:----|:-------|
| 累计收入/累计开票 | `incomes` |
| 累计成本 | `costs` |
| 累计回款 | `collections` |
| 累计付款 | `payments` |

---

### F3: 合同管理 KPI 不能跳转

**位置**: `ContractBusinessWorkbench.tsx:109-117`

7 个 KPI 全部使用 `message.info`。且页面本身没有 `onNavigate` prop。

**影响**: 合同页面看到所有经营汇总数据，但点击无反应。用户想从合同进入订单→必须手动切菜单。

**修复**: 
1. App.tsx 传入 `onNavigate` 
2. KPI 点击改为 `onNavigate`

| KPI | 应跳转 |
|:----|:-------|
| 合同数量 | `projects`（自身） |
| 合同总金额 | `projects` |
| 累计收入 | `incomes` |
| 累计成本 | `costs` |
| 累计利润 | `projects` |
| 累计回款 | `collections` |
| 累计付款 | `payments` |
