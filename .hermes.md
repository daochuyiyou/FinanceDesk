## 开发流程

**当前模式：Product Governance V1** — 详见 [Product_Governance_V1](Knowledge/Product_Governance_V1.md)

六阶段流程：Prototype → Checklist(FPS-1 七维度) → Review → Develop → Verify → Freeze
原则：**一页一 Sprint（One Page, One Sprint）** — 一个 Sprint 只做一个页面
唯一任务来源：**[Product Backlog](Knowledge/Product_Backlog.md)**（P0/P1/P2）
唯一验收标准：**[FPS-1 Product Standard V1](Knowledge/FPS-1_Product_Standard_V1.md)**（7 维度）

评审结果定义：**Approve** → Develop ｜ **Approve with P1** → Develop 带 TODO ｜ **Reject** → 返回 Prototype
→ 未获 Approve 的 Prototype，禁止进入 Develop

产品目录结构（每个页面的原型目录）：
```
prototypes/<sprint>-<page-name>/
├── index.html     → 静态原型
├── mock.json      → Mock 数据
├── checklist.md   → 产品检查项（Business→UX→UI 三维度）
├── decision.md    → 评审决策
└── freeze.md      → 冻结记录（含版本号）
```

冻结对象：**原型方案**（布局/交互以冻结原型为准，非布局类优化直接修改）
版本标识：`页面名称 V1.0 Frozen` → 布局调整升级为 `V1.1 Prototype` → 重新走全流程

产品检查三模板：
- `Knowledge/Product_Checklist_Template.md` — Business + UX + UI 三维度
