# Business Identity Standard — 业务编码体系标准

> **BDD-02A.2 P1 输出 · 永久文档（SSoT）**
> 更新时间：2026-07-05
> 本文件定义 FinanceDesk 全部业务编码规则。未经审批不得修改。

---

## 一、编码总则

| 规则 | 说明 |
|------|------|
| 每个业务对象有且仅有一个**业务编码**作为不可变身份标识 |
| 编码由**人工录入**或**ERP 导入**，系统不自动生成 |
| 编码一经创建不得修改 |
| 编码是跨系统（ERP ↔ FinanceDesk）的匹配键 |
| 编码使用**大写字母 + 数字 + 连字符**，禁止使用中文、空格、特殊字符 |

---

## 二、业务编码清单

### 2.1 合同编码（Contract Code）

| 属性 | 值 |
|------|-----|
| 字段 | `contract_no` |
| 格式 | `CT-{YYYY}-{NNN}`（示例：`CT-2026-001`） |
| 来源 | 人工录入 |
| 唯一性 | 全局唯一，UNIQUE 约束 |
| 创建后修改 | ❌ 禁止 |
| ERP 匹配 | ✅ ERP 项目编号 → `erp_no` 字段（独立于 `contract_no`） |
| Dashboard 使用 | 展示合同列表主键 |

### 2.2 订单编码（Order Code）

| 属性 | 值 |
|------|-----|
| 字段 | `order_no` |
| 格式 | 甲方订单编号，无强制格式要求（`ORD-2026-001` / `CG-001` / `甲方的编号`） |
| 来源 | **人工录入**（系统禁止自动生成） |
| 唯一性 | 全局唯一，UNIQUE 约束 |
| 创建后修改 | ❌ 禁止 |
| ERP 匹配 | ✅ **ERP 按 `order_no` 匹配**（BDD-02A.1 确认） |
| Dashboard 使用 | 订单列表主键，经营指标聚合键 |

### 2.3 成本供应商合同编码（Supplier Contract Code）

| 属性 | 值 |
|------|-----|
| 字段 | `contract_no`（`supplier_contract` 表） |
| 格式 | `SC-{YYYY}-{NNN}`（示例：`SC-2026-001`） |
| 来源 | 人工录入 / Excel 导入 |
| 唯一性 | 全局唯一，UNIQUE 约束 |
| 创建后修改 | ❌ 禁止 |
| ERP 匹配 | ❌ 不参与 ERP 匹配（成本供应商合同库为 Master Data） |
| Dashboard 使用 | 成本执行引用键 |

### 2.4 发票编码（Invoice Code）

| 属性 | 值 |
|------|-----|
| 字段 | `invoice_no`（`income_flow` 表） |
| 格式 | 发票号码，由税务系统定义 |
| 来源 | 人工录入 / ERP 导入 |
| 唯一性 | 不强制全局唯一（同号码不同订单可能，由 `order_id + invoice_no` 联合确定） |
| 创建后修改 | ❌ 禁止 |
| ERP 匹配 | ✅ ERP 开票按 `invoice_no` 匹配 |

### 2.5 收款凭证编码（Receipt Code）

| 属性 | 值 |
|------|-----|
| 字段 | `receipt_no`（`collection` 表） |
| 格式 | 银行/财务系统定义的凭证号 |
| 来源 | 人工录入 |
| 唯一性 | 不强制全局唯一 |
| 创建后修改 | ❌ 禁止 |
| ERP 匹配 | ✅ ERP 回款按凭证号匹配 |

### 2.6 付款凭证编码（Voucher Code）

| 属性 | 值 |
|------|-----|
| 字段 | `voucher_no`（`payment` 表） |
| 格式 | 银行/财务系统的支付凭证号 |
| 来源 | 人工录入 |
| 唯一性 | 不强制全局唯一 |
| 创建后修改 | ❌ 禁止 |
| ERP 匹配 | ✅ ERP 付款按凭证号匹配 |

### 2.7 ERP 项目编号（ERP Project Code）

| 属性 | 值 |
|------|-----|
| 字段 | `erp_no`（`project` 表） |
| 格式 | ERP 系统的项目编号 |
| 来源 | **ERP 导入**，只读 |
| 唯一性 | 不强制 FinanceDesk 唯一（ERP 侧保障） |
| 允许修改 | ❌ 只读 |
| 用途 | 合同侧记录 ERP 来源，用于交叉核对 |

## 三、编码规则汇总

| 业务对象 | 编码字段 | 来源 | UNIQUE | 创建后可修改 | ERP 匹配键 |
|---------|---------|:----:|:-----:|:----------:|:----------:|
| 合同 | `contract_no` | 人工 | ✅ | ❌ | 辅助（通过 erp_no） |
| 订单 | `order_no` | **人工** | ✅ | ❌ | **主要匹配键** |
| 成本供应商合同 | `contract_no` | 人工/Excel | ✅ | ❌ | ❌ |
| 收入流水 | `invoice_no` | 人工/ERP | ❌ | ❌ | ✅ |
| 收款 | `receipt_no` | 人工 | ❌ | ❌ | ✅ |
| 付款 | `voucher_no` | 人工 | ❌ | ❌ | ✅ |
| ERP 项目 | `erp_no` | **ERP 导入** | ❌ | ❌ 只读 | N/A |

---

## 四、禁止的编码实践

| ❌ 禁止 | 原因 |
|---------|------|
| 系统自动生成订单编号 | 订单编号由甲方定义，系统不得自动生成 |
| 修改已创建的编码 | 编码一旦创建即锁定 |
| 使用中文、空格、特殊字符 | 避免编码匹配歧义 |
| 同一编码在不同模块中含义不同 | 编码语义一致性 |
| 在 UI 中显示为可编辑的编码字段 | 创建后编码必须不可编辑 |

---

## 五、编码生命周期

```
编码创建（人工/ERP导入）
  → 编码锁定（不可修改）
  → 业务数据关联（使用编码引用）
  → 业务完成 → 编码归档（逻辑删除保留编码）
```
