# Business Analyzer Interaction — 经营分析器交互标准

> **PDD-02 P1 输出 · 永久文档（Product SSoT）**
> 更新时间：2026-07-06
> **FinanceDesk 全局 Business Analyzer 交互设计。不是普通筛选器，而是经营分析入口。**

---

## 一、产品定位

Business Analyzer 是 FinanceDesk **全局唯一经营分析入口**。

| 不是 | 是 |
|:-----|:----|
| 普通筛选器 (Filter) | **经营分析入口 (Business Analysis Entry)** |
| 页面级组件 | **全局状态** |
| 单向选择 | **分析路径导航** |

### 用户心智模型

```
用户操作：先选择分析范围
              ↓
系统反应：再展示经营数据
              ↓
用户解读：基于当前范围理解数据
```

---

## 二、四个分析层级

```
第一层：经营期间 (Period)
    ↓  默认：最近已导入月份（如 2026-06）
    ↓  支持：历史月份
    ↓  禁止：按日、按周

第二层：分析维度 (Dimension)
    ↓  标准值：公司 / 合同 / 项目 / 订单
    ↓  预留：客户 / 部门 / 负责人

第三层：经营对象 (Object)
    ↓  根据维度动态加载
    ↓  示例：维度=项目 → [崇左移动一期, 江州校园网, ...]

第四层：高级过滤 (Advanced Filter) — 默认折叠
        负责人 / 合同类型 / 合同状态 / 订单状态 / 收入状态 / 成本状态
```

---

## 三、交互规则

| 操作 | 反应 | 范围 |
|:-----|:-----|:------|
| 切换经营期间 | **全部数据刷新** | 全局 |
| 切换分析维度 | **经营对象自动刷新** | 全局 |
| 选择经营对象 | **Dashboard 自动刷新** | 全局 |
| 修改高级过滤 | **局部刷新** | 当前页面 |

### 顺序约束

```
① 经营期间 → ② 分析维度 → ③ 经营对象 → ④ 高级过滤 → ⑤ 数据刷新
                                                          ↑
                                                  不得改变顺序
```

---

## 四、全局状态规则 (PDD-004)

| 场景 | Analyzer 状态 |
|:-----|:--------------|
| 切换页面（Dashboard→合同→订单→收入→成本） | ✅ **保持** |
| 浏览器刷新 F5 | ✅ **保持**（localStorage） |
| 返回上一页 | ✅ **保持** |
| 重新登录 | ✅ **恢复**最近一次分析条件 |
| 新标签页打开 | ✅ 按 URL 参数恢复 |

|:-----|:--------------|

| **成本中心** | ✅ 必须 |

---

## 六、URL 状态

```bash
# 当前分析状态可通过 URL 参数表达
?period=2026-06&dimension=project&object=P001
```

| 参数 | 说明 | 示例 |
|:-----|:-----|:------|
| `period` | 经营期间 | `2026-06` |
| `dimension` | 分析维度 | `company` / `contract` / `project` / `order` |
| `object` | 经营对象 ID | `P001` |

---

## 七、交互不动点

| 规则 | 说明 |
|:-----|:------|
| **经营期间永远不为空** | 默认最近导入月份 |
| **分析维度始终可选** | Radio 按钮组 |
| **对象列表动态加载** | API 调用 |
| **高级过滤默认折叠** | 展开后才可见 |
| **所有变更自动触发刷新** | 无需点击"查询"按钮 |
