## 案例①：框架合同—收入导入

### 业务场景

企业级框架合同下，按订单分批开票，ERP 导入收入流水。

### 前置条件

| # | 条件 | SQL / API |
|:-:|------|-----------|
| 1 | 存在框架合同项目 | `Project(framework_name="中移建设2025框架合同")` |
| 2 | 存在子订单 | `Order(order_no="CG-2025-001", amount=500000)` |
| 3 | ERP Excel 包含该订单的收入流水 | `voucher_no="CG-2025-001", amount=100000` |

### 验收步骤

| 步骤 | 操作 | 预期结果 |
|:----:|------|---------|
| 1 | 上传 Excel | 解析完成，3 行有效数据 |
| 2 | Match Preview | 匹配成功 (P1: order_no) |
| 3 | Impact Preview | 收入 3 条，影响订单 1 个 |
| 4 | 确认导入 | Batch 创建成功 |
| 5 | **执行 Import Engine** | `POST /erp/engine/execute/{batch_no}` |
| 6 | 验证业务表 | `IncomeFlow` 新增 3 条，`invoice_no` 匹配 |
| 7 | 验证 Order Summary | 订单金额更新 |
| 8 | 验证 Revenu Summary | 收入汇总更新 |

### 验收数据

```csv
凭证号,发生时间,摘要,贷方金额,借方金额,项目名称
CG-2025-001,2025-03-01,框架合同第一期开票,100000,,中移建设
CG-2025-001,2025-06-01,框架合同第二期开票,200000,,中移建设
CG-2025-001,2025-09-01,框架合同第三期开票,200000,,中移建设
```

### 验收指标

| 指标 | 预期 | 实际 |
|:----:|:----:|:----:|
| 事务完整性 | 3 条全部写入 | — |
| 金额合计 | ¥500,000 | — |
| Invoice No | 3 个唯一值 | — |
| 订单金额更新 | 触发 Summary | — |

## 案例③：重复导入

### 业务场景

同一份 Excel 被错误上传并确认两次，Import Engine 必须正确处理。

### 前置条件

| # | 条件 |
|:-:|------|
| 1 | 案例①已完成导入 |
| 2 | 再次上传相同文件 |

### 验收步骤

| 步骤 | 操作 | 预期结果 |
|:----:|------|---------|
| 1 | 上传相同 Excel | 解析正常 |
| 2 | Impact Preview | duplicate_count > 0 |
| 3 | 确认导入 | 创建 Batch |
| 4 | 执行 Import Engine | 重复的记录标记为 `duplicate` 并跳过 |
| 5 | 验证业务表 | 无重复的 `invoice_no` |
| 6 | 验证 Order Summary | 金额没有被重复累加 |

### 验收指标

| 指标 | 预期 |
|:----:|:----:|
| duplicate_count | = 3 |
| 新增记录 | 0 |
| 业务表无重复 | ✅ |

## 案例④：红冲（冲红/负数）

### 业务场景

已开票的收入因退票/折扣需要红冲，ERP 中包含负数金额行。

### 前置条件

| # | 条件 |
|:-:|------|
| 1 | 存在订单 `Order(order_no="HC-2025-001")` |
| 2 | 原发票 `invoice_no="INV-2025-001"` 已存在（正数） |

### 验收步骤

| 步骤 | 操作 | 预期结果 |
|:----:|------|---------|
| 1 | 上传含负数金额的 Excel | 解析正常 |
| 2 | Impact Preview | 负数金额行正确识别 |
| 3 | 执行 Import Engine | 负数行写入 IncomeFlow（金额为负） |
| 4 | 验证 | Order Summary 金额正确（正+负=净值） |

### 验收数据

```csv
凭证号,发生时间,摘要,贷方金额,借方金额
HC-2025-001-R,2025-07-01,退还多收工程款,-50000,,
HC-2025-001,2025-07-01,重新开票正数,30000,,
```

### 验收指标

| 指标 | 预期 |
|:----:|:----:|
| 负数行正确导入 | IncomeFlow 含 -50000 |
| 净值 | -20000（-50000 + 30000） |
| Summary 更新 | 净额反映在汇总 |

## 案例⑤：人工匹配

### 业务场景

ERP 凭单号与系统订单编号不一致，需要通过 P4 人工确认后导入。

### 前置条件

| # | 条件 |
|:-:|------|
| 1 | 存在项目 `Project(framework_name="绿地中心项目")` |
| 2 | 存在订单 `Order(order_no="LD-2025-001")` |
| 3 | ERP 凭单号不等于 order_no（需人工匹配） |

### 验收步骤

| 步骤 | 操作 | 预期结果 |
|:----:|------|---------|
| 1 | 上传不匹配的 Excel | P1-P3 均失败 |
| 2 | Match Preview | 全部 P4 待确认 |
| 3 | 在 Workbench 中手动关联 | 关联到 `Order(id=LD-2025-001)` |
| 4 | 确认导入 | Batch 创建 |
| 5 | 执行 Import Engine | 写入 IncomeFlow，order_id 为人工选择的 |
| 6 | 验证 | 数据正确归属于指定订单 |

### 验收数据

```csv
凭证号,发生时间,摘要,贷方金额,借方金额
ERP-GREEN-001,2025-08-01,绿地中心工程款,150000,,
```

### 验收指标

| 指标 | 预期 |
|:----:|:----:|
| 人工匹配 | order_id = LD-2025-001 |
| Import Engine | 正确使用人工匹配结果 |
