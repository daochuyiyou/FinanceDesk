## 二、字段定义

### 2.1 基本信息组

| 字段 | 类型 | 必填 | 默认值 | 来源 | 校验规则 | 帮助说明 |
|------|:----:|:----:|:------:|:----:|---------|---------|
| framework_name | String(200) | ✅ | — | M | max_length=200, 不可空白 | 框架合同名称，唯一标识项目 |
| contract_no | String(100) | ✅(新增) | 自动生成 | M | UNIQUE, max_length=100 | 合同编号，新增时必填；历史数据可为空 |
| contract_type | Radio | ✅ | 框架合同 | M | 仅允许"单项合同""框架合同" | 合同业务类型 |
| framework_parent | Select | ❌ | — | A | 仅合同类型=框架合同 时显示 | 选择所属上级框架 |
| contract_year | Integer | ❌ | 取签订日期年份 | M/S | 4 位整数 | 所属年度，默认自动计算，允许修改 |
| owner_name | String(200) | ✅ | — | M | max_length=200, 不可空白 | 业主单位，支持后续 AutoComplete |
| owner_contact | String(100) | ❌ | — | M | — | 联系人 |
| owner_phone | String(50) | ❌ | — | M | 电话格式（可选） | 联系电话 |

### 2.2 合同信息组

| 字段 | 类型 | 必填 | 默认值 | 来源 | 校验规则 | 说明 |
|------|:----:|:----:|:------:|:----:|---------|------|
| sign_date | Date | ❌ | 今天 | M | ≤ today | 合同签订日期 |
| start_date | Date | ❌ | 今天 | M | ≤ end_date | 合同有效期开始 |
| end_date | Date | ❌ | 一年后 | M | ≥ start_date | 合同有效期结束 |
| project_type | Select | ❌ | 其他 | M | 字典值 | 项目类型（工程施工/维护服务等） |
| internal_or_external | Select | ❌ | 集团内 | M | 字典值 | 集团内外 |
| status | String(50) | ❌ | 待执行 | S/M | 字典值 | 合同状态：系统推导为主，人工仅可终止 |

### 2.3 甲方业主信息组

| 字段 | 类型 | 必填 | 默认值 | 来源 | 校验规则 | 说明 |
|------|:----:|:----:|:------:|:----:|---------|------|
| customer_name | String(200) | ❌ | — | M | max_length=200 | 甲方单位名称 |
| customer_contact_person | String(100) | ❌ | — | M | — | 甲方联系人 |
| customer_contact_phone | String(50) | ❌ | — | M | 电话格式 | 甲方联系方式 |

> **注**：甲方业主信息为新增概念。当前 `order` 表有 `customer_name` 字段，但 `project` 表无业主信息字段。需在 BDD-01 数据库变更中确认是否新增到 `project` 表，或通过 Order 关联。

### 2.4 ERP 信息组

| 字段 | 类型 | 必填 | 默认值 | 来源 | 校验规则 | 说明 |
|------|:----:|:----:|:------:|:----:|---------|------|
| erp_no | String(100) | ❌ | — | E | 只读 | ERP 项目编号，来自智慧工程平台 |

## 三、动态交互

### 3.1 合同类型切换

```
用户选择"单项合同"：
  → 隐藏 "所属框架" 字段
  → 清除 "所属框架" 值（如之前已选）
  → 显示 "合同编号" 为可选
  → 无框架关联

用户选择"框架合同"：
  → 显示 "所属框架" 字段
  → "所属框架" 变为可选（可新建框架或选择已有框架）
  → 合同属于该框架的子项目
  → "所属框架" 如果选择已有框架，自动填充框架的年度/有效期字段
```

### 3.2 显示/隐藏规则表

| 字段 | 单项合同 | 框架合同 | 说明 |
|------|:--------:|:--------:|------|
| 合同名称 | ✅ 显示 | ✅ 显示 | |
| 合同类型 | ✅ 显示 | ✅ 显示 | 单选切换 |
| 所属框架 | ❌ 隐藏 | ✅ 显示 | 动态控制 |
| 签订日期 | ✅ 显示 | ✅ 显示 | |
| 合同开始日期 | ✅ 显示 | ✅ 显示 | |
| 合同结束日期 | ✅ 显示 | ✅ 显示 | |
| 项目类型 | ✅ 显示 | ✅ 显示 | |
| 集团内外 | ✅ 显示 | ✅ 显示 | |
| 甲方信息 | ✅ 显示 | ✅ 显示 | |
| ERP 编号 | ✅ 显示（只读） | ✅ 显示（只读） | |

### 3.3 必填/可选动态规则

| 场景 | 字段 | 规则 |
|------|------|------|
| 框架合同 + 选择已有框架 | 合同开始日期 | 自动填充框架的开始日期（可修改） |
| 框架合同 + 选择已有框架 | 合同结束日期 | 自动填充框架的结束日期（可修改） |
| 框架合同 + 选择已有框架 | 所属年度 | 自动填充框架的年度 |

## 四、保存逻辑

### 4.1 保存流程

```
用户点击"保存"
  → 前端字段校验（必填/格式/长度）
  → 构建请求体 (ProjectCreate)
  → POST /api/v1/projects
  → 后端校验（project_id 唯一性等）
  → 写入数据库
  → 返回 ProjectResponse (含 id, created_at)
  → 跳转到合同详情页
```

### 4.2 保存后的系统行为

| 行为 | 规则 |
|------|------|
| 生成合同 | `project` 表创建一条记录，`is_deleted = false` |
| 允许创建订单 | 合同创建成功后，合同详情页显示"新增订单"按钮 |
| 禁止自动创建订单 | **不得在保存合同时自动创建订单**。订单创建需用户手动操作 |
| 合同状态 | `status` 字段：新合同默认为空或"待执行"（不强制要求状态字段） |
| 默认值 | 日期默认值："今天"；金额默认值：0；状态默认值："待执行" |

### 4.3 保存失败处理

| 失败场景 | 用户看到 |
|---------|---------|
| 必填字段未填 | 字段标红 + 提示信息 |
| 合同名称重复 | "该合同名称已存在" |
| 网络错误 | "保存失败，请重试" + 保留已填数据 |
| 服务器错误 | "系统异常，请联系管理员" + 保留已填数据 |

## 五、经营摘要

### 5.1 合同详情页顶部摘要卡片

```
┌──────────────────────────────────────────────────────────────────┐
│  经营摘要                                                  [ 编辑 ]│
│                                                                    │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐          │
│  │ 合同金额  │ │ 预算金额 │ │ 订单数量 │ │ 已收入   │ │ 已成本   │          │
│  │  ¥0.00   │ │  ¥0.00  │ │   0     │ │ ¥0.00   │ │ ¥0.00   │          │
│  └────────┘ └────────┘ └────────┘ └────────┘ └────────┘          │
│  ┌────────┐ ┌────────┐ ┌────────┐                                 │
│  │ 已回款   │ │ 已付款   │ │ ERP Gap │                                 │
│  │ ¥0.00   │ │ ¥0.00   │ │ ¥0.00   │                                 │
│  └────────┘ └────────┘ └────────┘                                 │
└──────────────────────────────────────────────────────────────────┘
```

### 5.2 摘要卡片字段定义

| 指标 | 数据来源 | 计算规则 | 当前值 |
|------|---------|---------|:------:|
| 合同金额 | ProjectBudget → SUM | 项目下所有预算金额合计 | 0 |
| 预算金额 | ProjectBudget → SUM | 项目下所有预算调整后金额 | 0 |
| 订单数量 | Order → COUNT | 项目下未删除的订单数 | 0 |
| 已收入 | IncomeFlow → SUM(order_id) | 项目下所有订单的收入开票合计 | 0 |
| 已成本 | CostFlow → SUM(order_id) | 项目下所有订单的成本合计 | 0 |
| 已回款 | Collection → SUM(flow_id→order_id) | 项目下所有订单的回款合计 | 0 |
| 已付款 | Payment → SUM(cost_id→order_id) | 项目下所有订单的付款合计 | 0 |
| ERP Gap | 系统计算 | 收入合计 - 回款合计 | 0 |

> **当前阶段允许全部为 0。仅冻结页面结构，待 BDD-07 经营驾驶舱实现后填充实际数据。**
