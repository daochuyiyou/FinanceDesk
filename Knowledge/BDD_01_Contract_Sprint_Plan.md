## 二、Feature 拆解

### F1 — 合同列表

| 维度 | 说明 |
|------|------|
| **说明** | 合同列表页（ProjectList.tsx）重新组织，按新菜单树命名为"合同中心"，展示框架合同和单项合同的列表 |
| **涉及页面** | `frontend/src/pages/ProjectList.tsx` |
| **涉及 API** | `GET /api/v1/projects`（已有，需确认分页、排序、筛选参数） |
| **数据库** | 无变更（现有 `project` 表） |
| **测试** | 后端：验证 GET list 分页+排序；前端：渲染测试 |
| **预估工时** | 4h（2h 后端确认 + 2h 前端调整） |
| **风险等级** | 🟢 低 — 现有功能基本稳定，主要是 UI 重命名和布局调整 |

**验收条件**：合同列表显示所有合同，支持分页，支持按状态/类型筛选。

---

### F2 — 合同新增

| 维度 | 说明 |
|------|------|
| **说明** | 新增合同表单（ProjectModal.tsx），区分框架合同/单项合同，含甲方业主信息字段 |
| **涉及页面** | `frontend/src/pages/ProjectModal.tsx` |
| **涉及 API** | `POST /api/v1/projects`（已有，需确认字段完整性） |
| **数据库** | 无变更 |
| **测试** | 后端：POST 创建+字段校验；前端：表单验证、提交试验 |
| **预估工时** | 4h（1.5h 后端确认 + 2.5h 前端表单） |
| **风险等级** | 🟡 中 — 需区分合同类型逻辑，需处理字典引用 |

**验收条件**：可创建框架合同和单项合同，必填字段有校验，字典字段可选。

---

### F3 — 合同编辑

| 维度 | 说明 |
|------|------|
| **说明** | 编辑已有合同信息。ERP 来源字段（erp_no）为只读 |
| **涉及页面** | `frontend/src/pages/ProjectModal.tsx`（编辑模式） |
| **涉及 API** | `PATCH /api/v1/projects/{id}`（已有） |
| **数据库** | 无变更 |
| **测试** | 后端：PATCH 更新+排除未传字段；前端：编辑模式回填 |
| **预估工时** | 3h |
| **风险等级** | 🟢 低 — 标准 CRUD 模式 |

**验收条件**：ERP 来源字段不可编辑，非 ERP 字段可正常更新。

---

### F4 — 合同详情

| 维度 | 说明 |
|------|------|
| **说明** | 合同详情页（ProjectDetail.tsx），展示合同全部信息 + 关联订单列表 + 预算概况 |
| **涉及页面** | `frontend/src/pages/ProjectDetail.tsx` |
| **涉及 API** | `GET /api/v1/projects/{id}`（已有）；`GET /api/v1/projects/{pid}/orders`（已有） |
| **数据库** | 无变更 |
| **测试** | 前端：详情渲染 + 订单列表嵌入式展示 |
| **预估工时** | 3h |
| **风险等级** | 🟡 中 — 详情页布局较复杂，需处理关联数据加载 |

**验收条件**：详情页展示完整合同信息，关联订单列表可跳转到订单中心。

---

### F5 — 框架合同支持

| 维度 | 说明 |
|------|------|
| **说明** | 框架合同的特殊逻辑：一个框架下可创建多个项目；列表页展示框架合同的所有子项目；项目信息中标明所属框架 |
| **涉及页面** | `ProjectList.tsx`、`ProjectModal.tsx`、`ProjectDetail.tsx` |
| **涉及 API** | 可能需新增 `GET /api/v1/projects?framework={name}` 筛选 |
| **数据库** | 无变更（现有 framework_name 字段已覆盖） |
| **测试** | 后端：框架下多项目创建；前端：框架筛选展示 |
| **预估工时** | 3h（1h 后端 + 2h 前端） |
| **风险等级** | 🟡 中 — 需要新增 API 查询参数，前端需调整列表展示 |

**验收条件**：创建项目时可选择"属于哪个框架"，框架合同下可查看所有关联项目。

---

### F6 — 单项合同支持

| 维度 | 说明 |
|------|------|
| **说明** | 单项合同的特殊逻辑：一个合同 = 一个项目，无需框架关联。清单项与框架合同同列表展示 |
| **涉及页面** | `ProjectList.tsx`、`ProjectModal.tsx` |
| **涉及 API** | 无变更 |
| **数据库** | 无变更 |
| **测试** | 创建单项合同 → 验证独立展示 |
| **预估工时** | 2h |
| **风险等级** | 🟢 低 — 已有实现，主要调整前端展示区分 |

**验收条件**：单项合同和框架合同在同一列表中以不同标识区分展示。

---

### F7 — Excel 导入

| 维度 | 说明 |
|------|------|
| **说明** | 合同批量导入功能：上传项目导入模板.xlsx，解析并写入 project 表 |
| **涉及页面** | `ImportModal.tsx`（已有组件）+ `DataImport.tsx` |
| **涉及 API** | `POST /api/v1/import/projects`（`data_import.py` 已有 ProjectRow 模型） |
| **数据库** | 无变更 |
| **测试** | 后端：模板解析 → 数据验证 → 写入 → 错误报告 |
| **预估工时** | 4h（2h 后端导入逻辑 + 1h 前端 + 1h 测试） |
| **风险等级** | 🟢 低 — 导入架构已实现，只需确保合同模板正确 |

**验收条件**：上传模板文件 → 校验字段 → 成功导入 → 返回导入报告。

---

### F8 — Excel 导出

**验收条件**：点击导出 → 下载 xlsx 文件 → 列头与导入模板一致 → 数据完整。

| 维度 | 说明 |

| **测试** | 前端：筛选条件组合测试 |
| **预估工时** | 2h |
| **风险等级** | 🟢 低 — 标准筛选模式 |

### F10 — 权限控制

| 维度 | 说明 |
|------|------|
| **说明** | 合同中心的页面访问权限：只有授权用户可查看/编辑/删除合同 |
| **涉及页面** | 所有合同中心页面（横切关注点） |
| **涉及 API** | 后端路由添加权限装饰器 |
| **数据库** | 可能需新增用户/角色表（BDD 阶段确认） |
| **测试** | 权限场景测试 |
| **预估工时** | 2h |
| **风险等级** | 🟠 较高 — 依赖用户认证体系，当前系统无独立用户管理 |

**验收条件**：未授权用户无法访问合同中心页面。

## 三、Sprint Backlog

### Sprint-1 建议安排

| 批次 | Features | 工时 | 说明 |
|:----:|----------|:----:|------|
| Batch 1 | F1 + F9 | 6h | 列表 + 筛选（前端为主） |
| Batch 2 | F2 + F5 + F6 | 9h | 新增+合同类型支持 |
| Batch 3 | F3 + F4 | 6h | 编辑+详情 |
| Batch 4 | F7 + F8 | 7h | 导入+导出 |
| Batch 5 | F10 | 2h | 权限 |
| **合计** | **10 Features** | **30h** | |

### 每个 Feature 的开发流程

```
Feature 开发入口
  → Build（代码修改）
  → Backend Tests（pytest）
  → Frontend Tests（Vitest）
  → Regression（全量测试）
  → API Contract 检查
  → UI Review
  → Business Rule Review
  → ✅ 审批通过 → 下一个 Feature
```
