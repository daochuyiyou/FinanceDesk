## 二、设计原则（永久冻结）

### PDD-001 — 经营期间优先

> **所有经营数据必须基于经营期间（Period）作为第一筛选条件。默认 = 最近已导入月份。**

| 属性 | 值 |
|:----:|-----|
| 默认期间 | 最近已导入月份（如 2026-06） |
| 数据范围 | 所有 KPI/趋势/分析均来自当前期间 |
| 切换 | 用户可切换期间，Dashboard 自动刷新 |

### PDD-002 — 多层分析顺序

> **分析顺序固定：经营期间 → 分析视角 → 分析对象 → Dashboard 自动刷新。不得反向。**

```
① 选择经营期间
       ↓
② 选择分析视角（Company / Contract / Project / Order）
       ↓
③ 选择分析对象
       ↓
④ Dashboard 自动刷新
```

## 四、核心概念

| 概念 | 说明 |
|:-----|:------|
| **经营期间** | 月度，如 2026-06 |
| **分析视角** | Company / Contract / Project / Order |
| **分析对象** | 当前视角下的具体对象 |
| **KPI** | 8 个核心经营指标 |
| **趋势** | 近 12 月逐月趋势 |
| **预警** | 经营异常自动检测 |
| **事项** | 待处理经营事项 |

---

## 五、禁止事项

| 禁止 | 原因 |
|:-----|:------|
| ❌ 展示今日数据 | 月度经营分析不需要日级数据 |
| ❌ 展示实时刷新 | 月度分析不需要实时 |
| ❌ 设计互联网 Dashboard 风格 | 面向工程项目管理，非互联网运营 |
| ❌ 跳过经营期间筛选 | 违反 PDD-001 |
| ❌ 跳过分析层次顺序 | 违反 PDD-002 |
| ❌ 直接展示 ORM 数据 | 必须通过 Summary Engine |
