# Next Action — 经营对象下一步动作

> **BDD-02B F3 业务冻结 · 暂不落库**
> 更新时间：2026-07-05

---

## 一、定义

Next Action（下一步动作）是系统根据经营对象的当前状态自动推导的"待办事项"。

**全部采用自动推导，禁止人工维护。**

## 二、推导规则

### 2.1 订单 Next Action

| 条件 | 下一步动作 | 优先级 |
|------|-----------|:------:|
| 订单创建后，无任何流水 | 等待录收入 / 等待录成本 | P0 |
| 有收入流水，无回款 | 等待收款 | P0 |
| 有成本流水，无付款 | 等待付款 | P0 |
| 回款 < 收入 | 继续收款 | P1 |
| 付款 < 成本 | 继续付款 | P1 |
| ERP 有同步数据但未匹配 | 等待 ERP 匹配 | P1 |
| 收入=回款 且 成本=付款 | ✅ 已完成 | — |
| 收入/回款/成本/付款数据异常 | ⚠️ 需要人工核对 | P0 |

### 2.2 合同 Next Action

| 条件 | 下一步动作 |
|------|-----------|
| 合同创建后，无订单 | 等待创建订单 |
| 有订单，订单未完成 | 跟踪订单执行 |
| 全部订单已完成 | ✅ 已完成 |

### 2.3 收入流水 Next Action

| 条件 | 下一步动作 |
|------|-----------|
| 刚创建，无回款 | 等待回款 |
| 部分回款 | 继续催款 |
| 已全部回款 | ✅ 已完成 |

### 2.4 成本流水 Next Action

| 条件 | 下一步动作 |
|------|-----------|
| 刚创建，无付款 | 等待付款 |
| 部分付款 | 继续安排付款 |
| 已全部付款 | ✅ 已完成 |

---

## 三、展示规则

| 规则 | 说明 |
|------|------|
| 展示位置 | 订单详情页 → Next Action 卡片（最显眼位置） |
| 颜色 | P0（红色）/ P1（橙色）/ ✅（绿色） |
| 禁止 | 人工修改、标记完成、添加自定义动作 |
