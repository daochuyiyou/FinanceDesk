## 三、冻结声明

### 3.1 页面架构已冻结

> **FinanceDesk UI Architecture 已进入冻结状态。**

自 2026-07-04 起，以下内容作为 UI SSoT 永久维护：

| 文档 | 冻结内容 |
|------|---------|
| `UI_Architecture.md` | 菜单树、12 个页面职责、权限矩阵 |
| `UI_Data_Source_Matrix.md` | 字段来源矩阵（M/E/S/A） |
| `Page_Flow.md` | 页面流转关系、导航规则、禁止项 |

### 3.2 冻结范围

| 维度 | 状态 | 说明 |
|------|:----:|------|
| 菜单结构 | 🔒 | 4 大分组，12 个主页面，4 个子页面 |
| 页面职责 | 🔒 | 维护/禁止维护数据边界 |
| 数据来源 | 🔒 | 字段级 M/E/S/A 矩阵 |
| 页面流转 | 🔒 | 收入侧/成本侧/ERP 集成/基础资料 4 流 |
| 导航规则 | 🔒 | 允许/禁止的页面跳转 |
| UI 样式/组件 | ✅ 可优化 | Ant Design 组件选型、布局样式 |

### 3.3 后续 BDD 页面开发必须遵循

1. **不得新增菜单项** — 除非出现新的业务域（需 Business Constitution 修改）
2. **不得给页面新增职责** — 超出当前边界属于范围蔓延
3. **不得将 Master Data 页面转为流程页面** — F007
4. **不得将 Dashboard 转为可写页面** — R005
5. **字段来源必须遵循矩阵定义** — R004
6. 页面流转必须遵循已定义的导航规则
