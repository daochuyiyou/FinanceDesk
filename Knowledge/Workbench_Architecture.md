# Workbench Architecture — 工作台架构

> **PDD-00 P5 输出 · 永久文档（Product SSoT）**
> 更新时间：2026-07-06
> **统一工作台规范。所有 Workbench 遵循统一布局和交互模式。**

---

## 一、Workbench 定义

Workbench 是 FinanceDesk 中**多步骤、高交互、带确认**的工作页面。

| 属性 | 值 |
|:----:|-----|
| 布局 | Steps + Card Content + Action Bar |
| 导航 | Steps 指示当前进度 |
| 数据 | 页面内管理，提交时持久化 |
| 安全 | 确认对话框不可跳过 |

---

## 二、Workbench 种类

| Workbench | 用途 | 步骤数 |
|:----------|:-----|:------:|
| **ERP Import Workbench** | Excel → 解析 → 匹配 → 影响 → 确认 | 4 |
| **Matching Workbench** | P4 人工匹配界面 | 2 |
| **Review Workbench** | 导入前审查 | 1 |
| **AI Workbench** (预留) | AI 分析交互 | 3 |
| **Approval Workbench** (预留) | 审批流程 | 2 |

---


## 四、各 Workbench 详解

### ERP Import Workbench

| 步骤 | 内容 | 已实现 |
|:----:|:-----|:------:|
| ① 上传 | Dragger 拖拽上传 Excel | ✅ |
| ② 解析 | 列映射 + 示例数据 | ✅ |
| ③ 匹配 | P1-P4 结果 + 人工匹配 | ✅ |
| ④ 影响预览 | 13 项 Dry Run 指标 | ✅ |
| ⑤ 确认 | Confirm Dialog + 导入结果 | ✅ |

### Matching Workbench

| 步骤 | 内容 |
|:----:|:------|
| ① 待匹配列表 | UNMATCHED / MULTI_MATCH 流水列表 |
| ② 人工选择 | 选择目标项目/订单 + 确认匹配 |

### Review Workbench

| 步骤 | 内容 |
|:----:|:------|
| ① 审查 | 导入预览、数据检查、确认或驳回 |

### AI Workbench（预留）

| 步骤 | 内容 |
|:----:|:------|
| ① 选择分析维度 | 时间/项目/订单范围 |
| ② AI 分析 | 等待 AI 分析完成 |
| ③ 结果展示 | 分析报告 + 建议 |

---

## 五、Workbench 通用规范

| 规范 | 说明 |
|:-----|:------|
| **Steps 不可跳过** | 必须按顺序完成 |
| **每步都有验证** | 未完成不可进入下一步 |
| **可取消** | 任何时候可取消返回 |
| **数据暂存** | 页面内管理，确认后持久化 |
| **响应式** | Steps 可收起为下拉（小屏） |
