## 1. 集成目标

与中移建设集团**智慧工程平台**对接，实现项目/订单/流水数据自动同步，减少手工录入工作量。

## 2. 架构原则

### 2.1 双轨架构（非侵入式）

```
ERP 系统 ──→ 暂存表（Staging）──→ ETL 清洗 ──→ 业务表
                    ↑
            手工录入 ──────────────→ 业务表（不受影响）
```

关键约束：
- **不修改现有业务表结构**
- **新数据走暂存表隔离**
- **业务表始终保持手工可写**
- 同步过程可重放、可追溯

### 2.2 数据流向

```
智慧工程平台 API Polling
         │
         ▼
    ERP 暂存表（staging_*）
         │
         ▼
    ETL 清洗引擎
         │
    ├── 模糊匹配（关键词匹配）
    ├── 去重（ON CONFLICT DO NOTHING）
    └── 对账聚合（合计校验）
         │
         ▼
    业务表（project / order / flow 等）
```

## 3. ETL 处理规范

### 3.1 数据清洗

| 步骤 | 说明 | 处理方式 |
|------|------|---------|
| 列映射 | 中文列名 → 英文字段 | `translate_headers()` |
| 日期标准化 | 多种日期格式 → ISO | `date.fromisoformat()` |
| 金额归一 | 符号判断分流 | 贷方→收入，借方→成本 |
| 去重 | 相同业务键去重 | `ON CONFLICT DO NOTHING` |
| 模糊匹配 | 对接 ERP 关键词 | 名称/编号模糊匹配 |

### 3.2 对账逻辑

- 导入完成后，暂存表合计与原始 Excel/ERP 合计比对
- 不一致则标记警告，不自动写入业务表
- 对账报告返回前端展示

## 4. 已知集成模块
