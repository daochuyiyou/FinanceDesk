## 三、合同类型与订单数量规则

### 3.1 核心规则

| 合同类型 | 允许订单数 | 规则 |
|---------|:----------:|------|
| **框架合同** | N 个 | 框架合同可创建任意数量订单 |
| **单项合同** | **1 个** | 单项合同最多只能创建 1 个订单 |

### 3.2 前端交互逻辑

```
用户选择关联合同 (project_id)
  → 后端/前端获取合同类型 (contract_type)
  → 判断合同类型

如果是"单项合同"：
  → 查询该合同已有订单数
  → 如果 ≥ 1 → 显示提示"单项合同仅支持一个订单，已存在订单"
  → 禁用保存按钮
  → 跳转到查看已有订单

如果是"框架合同"：
  → 正常显示新增表单
  → 无订单数量限制
```

### 3.3 后端校验

```python
# 创建订单时后端校验
order_count = db.scalar(
    select(func.count()).select_from(Order)
    .where(Order.project_id == project_id, Order.is_deleted == False)
)
project = db.get(Project, project_id)
if project.contract_type == "单项合同" and order_count >= 1:
    raise HTTPException(422, "单项合同仅允许创建一个订单")
```

## 四、保存逻辑

### 4.1 保存流程

```
用户点击"保存"
  → 前端字段校验
  → 后端校验合同有效性 + 订单数量规则
  → 生成 Order 记录
  → 返回 OrderResponse
  → 跳转到订单详情页
  → 显示"新增订单成功，可在此添加流水"
```

### 4.2 保存后的系统行为

| 行为 | 规则 |
|------|------|
| 生成订单 | `order` 表创建一条记录 |
| 合同订单数更新 | 合同台账中 `order_count` 自动 +1（下次查询时） |
| 禁止自动创建流水 | 不自动创建收入/成本流水 |
| 禁止自动创建收付款 | 不自动创建回款/付款 |
